Entry detail — redesign options

Static previews only (no diary.js / diary.css changes). Each phone uses the same fully filled record — all 7 form sections represented: species/title, sex & age, date/time, location + ground, weights (gralloch / clean / larder), shot block, notes, plus shooter / destination / weather.

Sample data: Fallow Doe · Yearling · Castle Acre · Woodland Block · .308 @ 153m · Heart/Lung · Client · Game dealer · weights 56/42/43 kg · weather OVC 7.8°C · 12 mph SE · 1005 hPa.

  • A — Overlap sheet: hero chips + 3×2 metrics beside photo; stacked field blocks (shot → notes → shooter → destination); place + ground; weather strip.
  • B — Short hero + full metric ribbon (all six weights/dist) + panel with KV rows for shot, notes, shooter, destination, ground, location, weather.
  • C — One card per concern (photo, weights grid, shot, shooter, destination, ground, location, notes, weather) — matches long filled forms.
  • D — Dashboard cards: when/where, weights tile row, shot & field, notes, weather.
  • E — Mono strip encodes the densest facts; then row list for every section including notes.
Option A
Editorial overlap

Rounded sheet overlaps the hero; facts beside thumbnail; single clear hierarchy. Good if you want less “form” and more “record card”.

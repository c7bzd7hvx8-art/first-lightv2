# Design preview / sandbox files

These are standalone HTML documents used to iterate on visual design
decisions before the chosen look shipped into `diary.html` /
`diary.css` / `index.html` / `styles.css`.

They are **not** referenced from any shipped page, **not** in the
service-worker precache list, and will not load on a production device.
Keep them for historical comparison — handy when reverting a visual
choice six months later.

If one becomes obsolete (e.g. banner-states-preview after the banner
system is rewritten), delete the file outright rather than leaving a
tombstone.
